# CGU 創意方法速查表

> 16 種創意方法，5 大類別，快速選擇適合的技法

---

## 🚀 發散類 (Divergent)

快速產生大量點子，不批判、不篩選

| 方法 | 說明 | 速度 | 適用層級 | 使用時機 |
|------|------|------|----------|----------|
| **mind_map** | 從中心概念向外放射擴展 | ⚡ Fast | L1, L2 | 需要整理思緒、建立概念架構 |
| **brainstorm** | 不批判的快速點子生成 | ⚡ Fast | L1-L3 | 需要大量點子、團隊發想 |
| **scamper** | 7種變形：替代、結合、調適、修改、他用、消除、重排 | ⚡ Fast | L1, L2 | 改良現有產品/概念 |
| **random_input** | 隨機詞強制聯想 | ⚡ Fast | L2, L3 | 卡關時、需要突破性思考 |

### SCAMPER 詳解

```
S - Substitute    替代：可以用什麼替代？
C - Combine       結合：可以和什麼結合？
A - Adapt         調適：可以從哪裡借鏡？
M - Modify        修改：可以放大/縮小/改變什麼？
P - Put to other  他用：可以用在其他地方嗎？
E - Eliminate     消除：可以去掉什麼？
R - Reverse       重排：可以顛倒/重新排列嗎？
```

## 🏗️ 結構類 (Structural)

系統化探索，確保全面覆蓋

| 方法 | 說明 | 速度 | 適用層級 | 使用時機 |
|------|------|------|----------|----------|
| **mandala_9grid** | 強制8方向聯想，可遞迴展開 | 🐢 Slow | L1, L2 | 深度探索單一主題 |
| **morphological** | 多維度選項的排列組合 | 🐢 Slow | L1, L2 | 系統性組合探索 |
| **5w2h** | What, Why, Who, When, Where, How, How much | ⚡ Fast | L1-L3 | 快速釐清問題/計畫 |
| **fishbone** | 6M 因果分析：人員、方法、材料、設備、測量、環境 | 🐢 Slow | L1, L2 | 根因分析、問題診斷 |

### 5W2H 詳解

```
What    - 是什麼？做什麼？
Why     - 為什麼？目的是？
Who     - 誰來做？給誰用？
When    - 什麼時候？時程？
Where   - 在哪裡？通路？
How     - 如何做？方法？
How much - 多少？成本？規模？
```

## 👁️ 視角類 (Perspective)

多元觀點，避免盲點

| 方法 | 說明 | 速度 | 適用層級 | 使用時機 |
|------|------|------|----------|----------|
| **six_hats** | 6種觀點：白(事實)、紅(情感)、黑(批判)、黃(樂觀)、綠(創意)、藍(統籌) | 🐢 Slow | L1-L3 | 全面評估、團隊決策 |
| **reverse** | 反向思考：如何製造問題？如何讓情況更糟？ | ⚡ Fast | L2, L3 | 發現盲點、預防問題 |
| **analogy** | 跨域類比：直接、擬人、符號、幻想類比 | 🐢 Slow | L2, L3 | 跨領域創新、突破框架 |

### 六頂思考帽詳解

```
⬜ 白帽 - 事實：數據是什麼？我們知道什麼？
🟥 紅帽 - 情感：感覺如何？直覺說什麼？
⬛ 黑帽 - 批判：風險是什麼？可能失敗的原因？
🟨 黃帽 - 樂觀：好處是什麼？最佳情況？
🟩 綠帽 - 創意：有什麼新想法？替代方案？
🟦 藍帽 - 統籌：結論是什麼？下一步？
```

## ⚙️ 系統類 (Systematic)

科學化的創新方法

| 方法 | 說明 | 速度 | 適用層級 | 使用時機 |
|------|------|------|----------|----------|
| **triz** | 40條發明原理 | 🐢 Slow | L1-L3 | 技術創新、專利迴避 |

### TRIZ 部分原理

```
1. 分割    - 將物體分成獨立部分
2. 抽取    - 只取出必要部分
3. 局部品質 - 使物體各部分執行不同功能
4. 非對稱   - 將對稱物體改為非對稱
5. 合併    - 在空間上合併同類物體
...
```

## 🎯 快速選擇指南

### 依情境選擇

| 情境 | 推薦方法 | 工具呼叫 |
|------|----------|----------|
| 「我需要很多點子」 | brainstorm | `generate_ideas(topic, count=20)` |
| 「我完全卡住了」 | random_input, reverse | `apply_method("reverse", topic)` |
| 「改良現有產品」 | scamper | `apply_method("scamper", product)` |
| 「需要全面評估」 | six_hats | `apply_method("six_hats", topic)` |
| 「要釐清問題」 | 5w2h | `apply_method("5w2h", problem)` |
| 「找問題根因」 | fishbone | `apply_method("fishbone", issue)` |
| 「跨領域創新」 | analogy, spark_collision | `spark_collision(domain_a, domain_b)` |

### 依創意層級選擇

```
Level 1 (組合創意)     → mind_map, brainstorm, 5w2h
Level 2 (探索創意)     → scamper, six_hats, mandala
Level 3 (變革創意)     → reverse, random_input, analogy
```

### 依時間選擇

```
5 分鐘  → brainstorm, 5w2h
15 分鐘 → scamper, reverse, mind_map
30 分鐘 → six_hats, mandala_9grid
1+ 小時 → double_diamond, design_sprint
```

## 🔗 方法組合技

### 發現問題 → 解決問題

```
1. 5w2h (釐清問題)
2. fishbone (分析根因)
3. reverse (反向思考)
4. brainstorm (發想解法)
5. six_hats (評估方案)
```

### 產品創新

```
1. scamper (變形現有產品)
2. spark_collision (跨域結合)
3. mandala_9grid (深度展開)
4. six_hats (全面評估)
```

### 突破卡關

```
1. reverse (製造問題法)
2. random_input (隨機刺激)
3. analogy (跨域類比)
4. deep_think(mode="spark")
```
